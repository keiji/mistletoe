# 用語集 (Thesaurus)

本書では、デザインドキュメントや開発プロセスで使用される主要な用語とその定義を記述します。

## 一般用語

| 用語 | 英語 | 説明 |
| :--- | :--- | :--- |
| **検証処理** | Verification Process | 入力値や環境状態が要件を満たしているか確認する一連の手順。エラーがある場合は処理を中断。 |
| **リポジトリ** | Repository | Gitによって管理されるプロジェクトのファイルと履歴の集合。 |
| **並列実行** | Parallel Execution | 複数のタスク（例：リポジトリのクローンやステータス確認）を同時に処理すること。これによる全体の処理時間の短縮。 |
| **競合** | Conflict | 複数の変更が同じファイルの同じ箇所で行われ、自動的に統合できない状態。または、リソースのロックなどが衝突すること。 |
| **マージツリー** | Merge Tree | `git merge-tree` コマンドの結果。実際にマージを行わずに、マージ結果の状態（競合の有無など）を確認するために使用。 |
| **設定** | Configuration | アプリケーションの動作を制御するための定義。通常は JSON ファイルで提供。 |
| **シャロークローン** | Shallow Clone | `git clone --depth` を使用して、履歴の一部（最新の数コミットなど）のみを取得するクローン方法。 |
| **Detached HEAD** | Detached HEAD | HEADがブランチの先端ではなく、特定のコミットを直接指している状態。 |
| **ベースブランチ** | Base Branch | プルリクエストやマージのターゲットとなるブランチ。 |
| **分岐** | Divergence | ローカルとリモートの双方が進行しており、単純な Fast-forward ができない状態。 |
| **Fast-forward** | Fast-forward | ブランチのポインタを単に前進させるだけのマージ操作。履歴が分岐しない。 |

## コマンド・機能関連

| 用語 | 英語 | 説明 |
| :--- | :--- | :--- |
| **スナップショット** | Snapshot | ある時点でのリポジトリの状態（URL, ブランチ, リビジョンなど）を記録した情報、またはそのファイル。 |
| **プルリクエスト** | Pull Request | コードの変更をリポジトリに取り込むよう提案する機能。GitHub などのプラットフォームで使用。 |
| **ステータス** | Status | リポジトリの現在の状態。変更の有無、リモートとの同期状態、競合の有無などを含む。 |
| **識別子** | Identifier | スナップショットのファイル名などに使用される、リポジトリの状態（URL, Branch/Revision）から算出されたハッシュ値。 |
| **Mistletoe ブロック** | Mistletoe Block | `pr create` コマンドによって PR 本文に挿入される、スナップショット情報や依存関係グラフを含む領域。 |
| **依存関係グラフ** | Dependency Graph | リポジトリ間の依存関係を定義した Mermaid 形式のグラフ。PR 間のリンク生成に使用される。 |

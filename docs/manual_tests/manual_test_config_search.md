# 手動テスト設計書: 親ディレクトリ設定探索

## 概要
カレントディレクトリに `mistletoe` の設定ファイルが存在しない場合、親ディレクトリを遡って設定ファイルを探索する機能を検証します。
また、探索範囲の制限や、見つかった設定ファイルのバリデーション動作も確認します。

## テストシナリオ

### シナリオ 1: 親ディレクトリ設定の検出
1.  親ディレクトリに `mistletoe.json` を配置します。
2.  子ディレクトリ（設定ファイルなし）に移動します。
3.  `mstl status` などのコマンドを実行します。
4.  親ディレクトリの設定ファイルが読み込まれ、コマンドが正常に実行されることを確認します。

### シナリオ 2: 設定ファイル未検出
1.  カレントディレクトリおよび全ての親ディレクトリに `mistletoe.json` が存在しない環境を用意します。
2.  `mstl status` を実行します。
3.  設定ファイルが見つからない旨のエラーが表示されることを確認します。

### シナリオ 3: 探索範囲の限界
1.  Gitリポジトリのルートから1つ上のディレクトリまで遡っても設定ファイルがない場合、それ以上探索せず、適切にエラーとなることを確認します。

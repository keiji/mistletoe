# 手動テスト設計書: PR カテゴリ分けロジック

## 概要
`mstl-gh pr create` コマンド実行時に、各リポジトリの状態に基づいて適切に処理カテゴリ（Push必要、PR作成必要、スキップなど）に分類されるロジックを検証します。
変更がないリポジトリ、Push済みのリポジトリ、未Pushのリポジトリなどが混在する環境で正しく動作することを確認します。

## テストシナリオ

### シナリオ 1: 変更なしリポジトリの除外
1.  変更もコミットもないリポジトリを含めます。
2.  `mstl-gh pr create` を実行します。
3.  当該リポジトリが処理対象から除外（スキップ）されることを確認します。

### シナリオ 2: 未Pushコミットがあるリポジトリ (Push and Create)
1.  ローカルにコミットがあるが、リモートにPushされていないリポジトリを用意します。
2.  `mstl-gh pr create` を実行します。
3.  当該リポジトリが「Repositories to Push and Create Pull Request」カテゴリに分類され、Push と PR作成が行われることを確認します。

### シナリオ 3: Push済みだがPR未作成のリポジトリ (Create No Push)
1.  ローカルのコミットは既にリモートにPushされているが、Pull Requestはまだ作成されていないリポジトリを用意します。
2.  `mstl-gh pr create` を実行します。
3.  当該リポジトリが「Repositories to Create Pull Request (No Push)」カテゴリに分類され、Pushなしで PR作成のみ行われることを確認します。

### シナリオ 4: 既知のPRがあるリポジトリ (Skip Create)
1.  既にPRが作成済みのリポジトリ（ローカルに変更なし）を用意します（※自動テストスクリプトではモック等でシミュレーション）。
2.  `mstl-gh pr create` を実行します。
3.  当該リポジトリに対して新規PR作成がスキップされることを確認します。

# 手動テスト設計書: Switch Remote Fallback

## 概要
`mstl switch` コマンドにおいて、指定されたブランチがローカルに存在しない場合でも、リモート（origin）に同名のブランチが存在すれば、自動的にFetchしてチェックアウトするフォールバック機能を検証します。

## テストシナリオ

### シナリオ 1: リモートブランチの自動チェックアウト
1.  ローカルには存在しないが、リモートには存在するブランチ（例: `feature/remote-only`）を用意します。
2.  `mstl switch feature/remote-only` を実行します。
3.  エラーにならず、リモートからブランチ情報が取得され、ローカルに新しくブランチが作成されてチェックアウトされることを確認します。
4.  `git branch -vv` 等で、Upstreamが正しく設定されていることを確認します。

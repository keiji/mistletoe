# 手動テスト設計書: 初期化時安全確認

## 概要
`mstl init` コマンドを実行するディレクトリに、管理外のファイルやディレクトリが存在する場合の安全性チェック機能を検証します。
予期しないファイルの上書きや混在を防ぐため、警告プロンプトが表示され、ユーザーの承認を求める動作を確認します。

## テストシナリオ

### シナリオ 1: 未知のファイルが存在する場合の警告
1.  任意のファイル（例: `garbage.txt`）を含むディレクトリを用意します。
2.  そのディレクトリで `mstl init` を実行します。
3.  「未知のファイルが存在する」旨の警告が表示され、続行確認のプロンプト（[yes/no]）が出ることを確認します。

### シナリオ 2: ユーザーによる拒否
1.  シナリオ1と同様に警告を表示させます。
2.  プロンプトに対して `no` を入力します。
3.  初期化処理が中断され、変更が加えられないことを確認します。

### シナリオ 3: ユーザーによる承認
1.  シナリオ1と同様に警告を表示させます。
2.  プロンプトに対して `yes` を入力します。
3.  初期化処理が続行され、完了することを確認します。

### シナリオ 4: --yes フラグによる強制実行
1.  シナリオ1と同様の環境を用意します。
2.  `mstl init --yes` を実行します。
3.  警告メッセージは表示されるものの、プロンプトでの確認は行われず、自動的に初期化が実行されることを確認します。

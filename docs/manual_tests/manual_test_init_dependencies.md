# 手動テスト設計書: 初期化時依存関係検証

## 概要
`init` コマンド実行時に `--dependencies` フラグを指定した際の挙動を検証します。
依存関係設定が正しくコピーされるケース、指定されたIDが不正なケース、指定されたファイルが存在しないケースを網羅します。

## テストシナリオ

### シナリオ 1: 正常系 - 依存関係設定のコピー
1.  依存関係を定義した設定ファイル (`mistletoe.json` 相当) を用意します。
2.  `mstl init --dependencies <config_file>` を実行します。
3.  初期化されたディレクトリ内の `mistletoe.json` に、元の設定ファイルの依存関係情報が正しく反映されていることを確認します。

### シナリオ 2: 異常系 - 不正な依存関係 ID
1.  存在しないリポジトリIDを含む依存関係設定ファイルを用意します。
2.  `mstl init --dependencies <config_file>` を実行します。
3.  エラーが発生し、初期化が中断される、または適切な警告が表示されることを確認します。

### シナリオ 3: 異常系 - 依存関係ファイル欠損
1.  `--dependencies` フラグで存在しないファイルパスを指定します。
2.  `mstl init --dependencies <non_existent_file>` を実行します。
3.  エラーが発生し、初期化が中断されることを確認します。

# 手動テスト設計書: PR作成時 Behind 検知

## 概要
`mstl-gh pr create` コマンドを実行する際、ローカルブランチがリモートブランチ（Push先）よりも遅れている（Behind）、または分岐している（Diverged）場合に、処理を中断してエラーとする安全機構を検証します。
これにより、古い状態からの強制Pushや意図しない上書きを防ぎます。

## テストシナリオ

### シナリオ 1: Behind 状態での検知
1.  ローカルブランチをリモートの HEAD よりも古いコミットに戻します（`git reset --hard HEAD~1` 等）。
2.  `mstl-gh pr create` を実行します。
3.  「リモートより遅れている」旨のエラーが表示され、PR作成処理が中断されることを確認します。

### シナリオ 2: Diverged 状態での検知
1.  ローカルでコミットを作成し、リモートでも別のコミットが作成されている状態（分岐）を作ります。
2.  `mstl-gh pr create` を実行します。
3.  「分岐している（コンフリクトの可能性がある）」旨のエラーが表示され、処理が中断されることを確認します。

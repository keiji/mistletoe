# 手動テスト設計書: PR作成時 Base Branch 欠損

## 概要
Pull Request の作成先となる Base Branch（例: `main` や `develop`）がリモートリポジトリに存在しない場合、そのリポジトリに対する `pr create` 処理をスキップする動作を検証します。
これにより、存在しないブランチへのPR作成試行によるエラーを防ぎます。

## テストシナリオ

### シナリオ 1: Base Branch 欠損時のスキップ
1.  設定ファイルで `base_branch` として `non-existent-base` を指定したリポジトリを用意します（または、実際のリモートでそのブランチを削除します）。
2.  `mstl-gh pr create` を実行します。
3.  対象のリポジトリについて「Base Branch が見つからないためスキップする」旨の警告ログが出力され、エラーによる停止はせずに次のリポジトリの処理へ進むことを確認します。

### シナリオ 2: 正常なリポジトリとの混在
1.  Base Branch が存在するリポジトリと、存在しないリポジトリを混在させます。
2.  `mstl-gh pr create` を実行します。
3.  Base Branch が存在するリポジトリについては正常にPRが作成され、存在しないリポジトリのみがスキップされることを確認します。

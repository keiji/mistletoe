# 手動テスト設計書: 親ディレクトリ設定スイッチ

## 概要
親ディレクトリの設定ファイルが検出された場合、アプリケーションの実行コンテキスト（ワーキングディレクトリなど）が親ディレクトリに切り替わることを検証します。
これにより、サブディレクトリからコマンドを実行しても、ルートディレクトリで実行したのと同じ結果が得られることを保証します。

## テストシナリオ

### シナリオ 1: ワーキングコンテキストの切り替え確認
1.  親ディレクトリをルートとする `mistletoe` 環境を用意します。
2.  サブディレクトリに移動します。
3.  `mstl status` を実行します。
4.  出力結果が、サブディレクトリだけではなく、親ディレクトリ配下の全リポジトリの状態を反映していることを確認します。

### シナリオ 2: パス解決の整合性
1.  相対パスで指定されたリポジトリを含む設定ファイルを親ディレクトリに配置します。
2.  サブディレクトリからコマンドを実行します。
3.  リポジトリのパスが、(サブディレクトリ基準ではなく) 親ディレクトリ基準で正しく解決され、操作が行われることを確認します。
